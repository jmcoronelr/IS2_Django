content.views

Funciones

    agregar_comentario

    Parámetros: (request, pk)

    Retorno: None

    Agrega un comentario a un contenido específico. Verifica si el formulario de comentario es válido y lo guarda en la base de datos.

    cambiar_estado_contenido

    Parámetros: (request, pk)

    Retorno: None

    Cambia el estado de un contenido entre 'published' e 'inactive'. Muestra un mensaje de éxito indicando el nuevo estado del contenido.

    content_create_edit

    Parámetros: (request, pk=None)

    Retorno: None

    Crea o edita un contenido. Si se proporciona una clave primaria (pk), se edita el contenido existente; de lo contrario, se crea uno nuevo. Solo los superusuarios pueden ver todas las categorías; los demás usuarios ven categorías según sus roles. Maneja la carga de bloques de contenido dinámicos y actualiza el historial de cambios del contenido.

    content_delete

    Parámetros: (request, pk)

    Retorno: None

    Elimina un contenido seleccionado. Permite eliminar un contenido confirmando la acción con el usuario.

    content_detail

    Parámetros: (request, pk)

    Retorno: None

    Muestra los detalles de un contenido específico, incluidas las interacciones del usuario. También permite agregar comentarios al contenido y verificar si el usuario ha dado like o dislike.

    content_list

    Parámetros: (request)

    Retorno: None

    Muestra una lista de contenidos, con la opción de filtrarlos por permisos, estado, categoría, fecha y título. Si el usuario es superusuario, se muestran todos los contenidos. Los usuarios regulares solo ven los contenidos que corresponden a las categorías en las que tienen roles con permisos específicos. También se incluye la opción de paginar los resultados.

    dislike_content

    Parámetros: (request, pk)

    Retorno: None

    Maneja la lógica de 'dislike' en un contenido. Si el usuario ya ha dado 'dislike', lo elimina; si no, lo agrega. Si el usuario había dado 'like', también se elimina.

    get_plantilla_blocks

    Parámetros: (request, plantilla_id)

    Retorno: None

    Obtiene los bloques asociados a una plantilla específica. Devuelve un JSON con los detalles de los bloques, como tipo, texto, posición y tamaño.

    like_content

    Parámetros: (request, pk)

    Retorno: None

    Maneja la lógica de 'like' en un contenido. Si el usuario ya ha dado 'like', lo elimina; si no, lo agrega. Si el usuario había dado 'dislike', también se elimina.
